Trace Privacy Policy

Last updated: June 14, 2025

Trace is a Chrome Extension designed to work entirely client-side. It does not collect, store, transmit, or process any personal or usage data externally. We believe in building software that respects user privacy by design.

1. No Data Collection

Trace does not collect or transmit any personal information.

  • There is no backend. No servers. No tracking scripts inside the extension.
  • All clips, stashes, and user content are stored locally in your browser.
  • We do not use cookies, fingerprinting, or third-party scripts in the extension.

2. Permissions

Trace requests only the permissions necessary for its single purpose: enabling fast, reflexive clipboard management and memory for knowledge workers.

  • storage – to save your clips and stashes locally.
  • scripting – to inject the command palette or tooltip when triggered by you.
  • clipboardWrite – to allow copying clips to your clipboard on demand.
  • host permissions – to allow Trace to run when activated by the user (e.g., typing ;; or using the popup).

None of these permissions are used to read page content or pass any data off-device.

3. Analytics Disclosure

Trace does not use analytics inside the extension.

The Chrome Web Store listing may include Google Analytics only on the listing page (as managed by the Chrome Web Store platform itself). This is outside our extension runtime and does not reflect any tracking behavior of the extension.

4. Children’s Privacy

Trace is not intended for users under the age of 13. No data is collected from any user, and Trace is compliant with COPPA by design.

5. Your Rights

Since we do not collect any personal data, there is no personal information to access, export, or delete.

However, if you have questions, you can contact us at: shreyanshpandey@protonmail.com

6. Changes to This Policy

We may update this policy if features change. All changes will be posted on the extension listing and within the extension’s changelog if relevant.
